This article covers the start of the “shellshock” bug. This bug is one of the oldest in history and it all started with a software program called Bash.

  • What is Bash?
  • It is a shell utility. It is a way of “talking” with the operating system that predates the graphical user interface. (GUI)
  • Quick and easy way to connect to computer systems.
  • Simple way for engineers to put web software on the OS

  • Is Bash still used today?
  • Yes, it is on the Mac and anyone who runs the Linux OS (as of this article, today it is more prevalent.)

  • Who built it and why was Bash created?
  • It was created by Brian Fox and it is a software program that was built for the UNIX OS
  • Bash was tagged with a license that let anyone use the code and redistribute it to anyone.
  • Fox hung around Richard Stallman and they both believed in the Free Software movement
  • Free Software Movement- create software that was free hackable and didn't have any copy restrictions. The idea to rebuild UNIX and share it.

  • What does shellshock have to do with bash?
  • In 1992 an engineer typed a bug into the code. Chet Ramey thinks he wrote the code. Ramey took over from Brian Fox as the lead developer of Bash in the late 1980s.
  • 20 years later the bug was discovered and named Shellshock. Officially taking the title of “oldest known unpatched bug.”
  • It was a security vulnerability and hackers released the code that could take over machines and turn them into malicious botnets. Result of the open-source internet, hackers could use the old code.

  • Has anything like this happened since?
  • Yes, another one was found after Shellshock in 2014 called Heartbleed. That was discovered on another open-source project called OpenSSL.

  • Why does this happen? Doesn’t Bash and other open-source programs have security?
  • According to this article, “Like the OpenSSL software, Bash has never had a full blown security audit, and it’s developed by a skeleton crew with virtually no financial support, That, unfortunately is the story of the internet.”

  • Why does open-source software choose not to have security audits?
  • One of the major ideas of open-source is that millions of people can view the code therefore, the bugs get fixed more quickly than other kinds of software, where code is hidden.  This is called Linus’ Law. named after Linus Torvald, who created the Linux operating system.

  • Does Linus’ Law work?
  • According to Robert Graham, CEO of Errata Security, not that many eyes get on the code at all. He states, “There’s a lot of code that doesn’t actually get very many eyes at all, and a lot of open-source projects don’t actually have all that many developers involved, even when they are fairly core.”

  • Doesn’t this happen with other software, no matter if it’s open-source?
  • Yes, Microsoft has faced security issues and spent millions to make security audits a priority

  • Is Bash working on security?
  • As of this article, Linux is working on security for open sources projects, like OpenSSL, OpenSSH and the Network Time Protocol. Bash isn’t on the list.
